# AppSheet DocGen Pro (High-Fidelity Document Generation)

## Description

This Google Apps Script project addresses a common limitation within the **AppSheet** no-code platform.

While AppSheet has a built-in document generation engine, it does not natively support generating editable Google Docs. Furthermore, the PDFs generated by AppSheet, though functional, can sometimes deviate from the original template's aesthetics, especially with complex layouts.

**AppSheet DocGen Pro** bridges this gap. This script enables the generation of either **Google Docs** or **high-fidelity PDFs** from a Google Docs template, scrupulously respecting the template's original formatting. It acts as a premium rendering engine for your AppSheet ecosystem.

## Features

- ✅ **Template Fidelity:** Strict adherence to the original Google Docs template formatting (fonts, styles, tables, images).
- ✅ **Multi-Format Output:** Generate in Google Docs format (editable) or PDF.
- ✅ **Advanced AppSheet Integration:** The script analyzes your data structure and performs a series of API requests to AppSheet to dynamically compile all data required for template evaluation.
- ✅ **Image Support:** Supports images (Image, Photo, Drawing, Signature, Thumbnail) sourced from AppSheet.

## Google Services Used

- **Google Drive:** For file and folder management, and PDF conversion.
- **Google Docs API:** For template manipulation and rendering.
- **Google Apps Script URL Fetch Service:** To communicate with the AppSheet API.

## Prerequisites

Before configuring the script, ensure you have the following:

1.  **Script Access:** Copy all script files into a Google Apps Script project accessible by the account that owns the AppSheet application.
2.  **Structure Google Sheet:** Create a Google Sheet with a tab containing these headers: `table`, `column`, `type`, `source`. This Sheet will map your AppSheet data model.
    *   **table:** Name of the AppSheet table.
    *   **column:** Name of the column.
    *   **source:** Name of the source table if the column is a reference.
    *   **type:** Column type (choose from: `key`, `ref`, `ref_list`, `image`).
        *   *Note 1: An `EnumList` of Ref type pointing to a table "item" should be set as type `ref_list` and source `item`.*
        *   *Note 2: Only columns of type `key`, `ref`, `ref_list`, or `image` need to be included in this Sheet.*
        *   *Note 3: The `image` type encompasses AppSheet's Image, Photo, Drawing, Signature, and Thumbnail types.*
3.  **Google Docs Template:** Create a template using standard AppSheet automation template syntax (e.g., `<<[ColumnName]>>`).
4.  **Destination Folder:** Create a Google Drive folder to store all documents generated by the script.

## Installation and Configuration

### Security Note regarding Key Storage

**⚠️ Important Security Warning:** This reusable version is designed *not* to store any sensitive keys or identifiers within the code. This is a deliberate choice for portability.

However, in a production environment, you **must secure** sensitive information (`APP_ACCESS_KEY`). We strongly recommend using the Apps Script `PropertiesService` (`ScriptProperties`) rather than passing them as plain text via call parameters.

### TEST Mode (Manual Execution)

To quickly test the script within the Apps Script editor:

1.  Open the `test_main` function.
2.  Modify it to include your actual identifiers:
    *   `APP_ID`
    *   `APP_ACCESS_KEY`
    *   `APPSHEET_REGION`
    *   `template_doc_id`
    *   `structure_ss_id` (The ID of the prerequisites Sheet)
    *   `folder_id`

### PRODUCTION Mode (AppSheet Automation)

To integrate the script into an AppSheet Automation:

1.  Create a 'Call a script' task within an AppSheet Automation bot.
2.  Configure the task to call the Apps Script function `main_call_template`, passing the required parameters (IDs, keys, output format, etc.).
